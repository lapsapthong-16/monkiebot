Turning dense PDFs into answerable knowledge.

This paper presents an intelligent document interaction system: upload an academic PDF, retrieve the most relevant passages with vector search, then use a large language model to answer questions with measurable quality.

The Problem

LLMs are fluent, but long documents expose their limits.

Academic PDFs are long, technical, and multi-topic. A plain chatbot can miss details, exceed context limits, or answer from general memory. The paper addresses this by separating the task into retrieval first, generation second.

Extract

Convert the PDF into clean text so the system can work with the document rather than a static file.

Segment

Split the paper into overlapping chunks to keep concepts intact while making retrieval precise.

Embed

Represent each chunk numerically using sentence embedding models trained for semantic similarity.

Retrieve

Use FAISS and cosine similarity to find the chunks that best match the reader's question.

Generate

Pass the retrieved context to an LLM so answers are grounded in the uploaded document.

Evaluate

Measure not only word overlap, but semantic match, naturalness, and response speed.

Main finding

There is no single universal winner. The best configuration depends on whether the product needs speed, semantic precision, or human-like fluency.

Best speed-quality balance

Gemma 2-9B IT with all-MiniLM-L6-v2 achieved perfect ROUGE-1 in the study while remaining fast enough for real-time interaction.

Best semantic relevance

Mixtral 8x7B with all-mpnet-base-v2 produced the highest BERTScore, making it compelling when meaning-level precision matters most.

Evaluation Design

The paper argues that “good” answers need more than one metric.

A useful document assistant should quote the right ideas, preserve the meaning of the source, sound natural to a reader, and respond quickly enough to feel usable.

R1

ROUGE-1 checks lexical overlap.

It rewards answers that share important words with the expected answer, but can miss meaning when phrasing changes.

BS

BERTScore checks semantic similarity.

It looks beyond exact words and evaluates whether the generated answer captures the same meaning.

HLP

Human-like performance checks usefulness.

The proposed HLP metric adds a human-centered lens: fluency, relevance, naturalness, and contextual alignment.

Portfolio Takeaway

The project is not just a chatbot. It is a measured document intelligence pipeline.

For hiring managers, the work demonstrates applied AI engineering: PDF processing, retrieval architecture, embedding model selection, LLM orchestration, and evaluation design.

For technical readers, the key contribution is the controlled comparison of retrieval and generation configurations, showing concrete trade-offs between accuracy, semantic quality, human-like output, and latency.

For non-technical visitors, the idea is simple: make long research documents conversational, searchable, and easier to understand without losing the source context.
